# Hank's Tank Backend

> **Production API:** https://hankstank.uc.r.appspot.com  
> **Companion repos:** [hanks_tank](../hanks_tank) Â· [hanks_tank_ml](../hanks_tank_ml)

This repo is the TypeScript/Express backend for Hank's Tank, an MLB analytics platform that combines historical warehouse queries, live MLB data, scheduled prediction outputs, and precomputed scouting reports behind a single API.

## What this repo does

- Serves historical team, player, standings, transactions, and Statcast data
- Routes requests between BigQuery-backed historical data and live MLB API reads
- Exposes daily prediction rows and scouting-report payloads generated by the ML pipeline
- Schedules pregame work through App Engine endpoints and Cloud Tasks
- Archives selected news content to BigQuery for downstream use

## Architecture

```text
React SPA / browser clients
    |
    +-- GET /api/*
           |
           +-- Express + TypeScript
                 |
                 +-- BigQuery
                 |     - mlb_2026_season
                 |     - mlb_historical_data
                 |
                 +-- MLB Stats API
                 +-- FanGraphs
                 +-- News API
                 +-- Cloud Tasks / scheduler-triggered pregame flows
```

## Key API areas

| Area | Representative routes |
|---|---|
| Health | `GET /health` |
| Predictions | `GET /api/predictions`, `GET /api/scouting-reports` |
| Stats | `GET /api/teamBatting`, `GET /api/teamPitching`, `GET /api/standings` |
| Players / teams | `GET /api/player/:playerId`, `GET /api/team/:teamAbbr` |
| Statcast | `GET /api/statcast` |
| Transactions | `GET /api/transactions`, `GET /api/team/:teamAbbr/transactions` |
| News | `GET /api/mlb-news`, `GET /api/braves-news`, `POST /api/news/refresh` |

## Production footprint

| Resource | Notes |
|---|---|
| Hosting | Google Cloud App Engine `default` service |
| Warehouse | BigQuery datasets for current-season and historical MLB data |
| ML integration | Reads `game_predictions`, `game_scouting_reports`, and feature-derived fields produced by `hanks_tank_ml` |
| Scheduler integration | Supports pregame orchestration via `/api/lineup/schedule-today` and Cloud Tasks |
| CI/CD | GitHub Actions test/build workflow with deploy-on-main |

## Local development

1. Install dependencies:
   ```bash
   npm install
   ```
2. Create `.env` from `.env.example`.
3. Set required environment variables:
   ```env
   GOOGLE_CLOUD_PROJECT=hankstank
   GOOGLE_APPLICATION_CREDENTIALS=C:\path\to\service-account.json
   NEWS_API_KEY=your-key
   PORT=8080
   ```
4. Start the app:
   ```bash
   npm run dev
   ```

## Useful scripts

| Command | Purpose |
|---|---|
| `npm run dev` | Compile and start the API locally |
| `npm run type-check` | Run TypeScript without emitting |
| `npm test` | Run Jest tests |
| `npm run build` | Compile to `dist/` |
| `npm run deploy` | Build and deploy to App Engine |
